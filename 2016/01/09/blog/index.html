Windows下使用Hexo搭建个人博客

发表于 2016-01-09   |   分类于 其他

一. 博客分类

  1. 托管博客：CSDN，博客园 等；
  2. 私有博客：WordPress，Hexo 等；

二. 安装相关软件

  1. 安装Node.js (Hexo 基于 Node.js 开发)
  2. 安装Git
  3. 安装文本编辑器 Sublime Text 或UltraEdit (可选)

三. 配置GitHub

  • 登陆GitHub官网，注册GitHub账号；
  • 创建仓库Repository
    仓库名必须为 your_user_name.github.io

  • 配置SSH-key

    • 设置用户名和邮箱（可选）
      • <git config --global user.email "{Email}">
      • <git config --global user.name "{UserName}">
    • 检查Git是否存在ssh-key
      • <$ cd ~/. ssh> 检查本机的ssh密钥
    • 创建.ssh文件
      • <$ mkdir ~/.ssh>
      • <$ cd ~/. ssh>

    • 生成密钥

      • $ ssh-keygen -t rsa -C {your_email@sample.com}

      • 命令执行成功后，.ssh目录下会生成id_rsa和id_rsa.pub文件

    • 配置密钥
      • 打开id_rsa.pub文件，复制其内容粘贴到GitHub的SSH中
        { Account settings -> SSH Keys -> Add SSH Key }
    • 验证密钥
      　　$ ssh -T git@github.com
      　　如果是下面的警告：
      　　The authenticity of host 'github.com (207.97.227.239)' can't be established.
      　　RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
      　　Are you sure you want to continue connecting (yes/no)?
      　　输入yes回车
  • 可选GitCafe，同GitHub类似配置

四. 安装Hexo

  • 安装
    • 打开Git bash
      $ npm install hexo –g
      （注：NPM的全称是Node Package Manager，是一个Node.js包管理和分发工具，已经成为了非官方的发布Node模块（包）的标准。）
    • 已安装Hexo，更新Hexo
      $ npm update hexo –g
  • 配置本地博客
    • 创建目录
      $ mkdir Hexo # 创建的目录名称和盘符位置可变
    • 初始化Hexo空间
      $ cd Hexo
      $ Hexo init
    • 安装node_modules
      $ npm install
      $ npm install hexo-deployer-git –save
    • 生成静态页面
      $ hexo generate
    • 启动服务预览
      $ hexo server
  • 至此本地部署已完成，浏览器输入 http://localhost:4000 就可以预览效果；

五. 部署到GitHub

  • 修改_config.yml文件
    deploy:
    type: git
    repository: git@github.com:liman/liman.github.io.git
    branch: master
    如果是为一个项目制作网站，那么需要把branch设置为gh-pages
  • 开始部署
    $ hexo deploy
  • 验证部署
    浏览器输入your_user_name.github.io

六. 部署总结

  • 每次部署，可按步来进行
    $ hexo clean #清除缓存
    $ hexo generate
    $ hexo deploy
  • 本地调试
    $ hexo g #生成静态网页
    $ hexo s #启动本地服务，进行文章预览调试
    浏览器输入http://localhost:4000 查看搭建效果

七. 更新主题

  • 克隆主题
    $ cd themes
    $ git clone https://github.com/iissnan/hexo-theme-next themes/next
  • 启用主题
    克隆/下载 完成后，打开 站点配置文件，找到 theme 字段，并将其值更改为 next。
  • 验证主题
  • 部署主题
    $ cd themes/next
    $ git pull origin master
    $ hexo d -g

八. 编写文章

  • 创建新文章
    $ cd your-hexo-site
    $ hexo new post "我的博文" / 创建一个“我的博文”的md文件。 /
    命令执行完成，在Hexo/sources/_post下生成一个md文件
  • md文件属性
    • Front-matter定义了文件属性
      • title：定义了博文的标题
      • date：定义了创作此博文的时间
      • tags：定义了博文的标签
        除了这三个属性以外我们还可以扩展设置一些属性。
      • update：定义了最后修改的时间
      • comments：定义能否评论此博文（true/false，默认为true）
      • categories：定义了博文的种类
  • Markdown语法
    自行百度或Google
    参照Markdown在线编辑器

九. 常用命令

  • hexo new "postName" #新建文章
  • hexo new page "pageName" #新建页面
  • hexo generate #生成静态页面至public目录
  • hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
  • hexo deploy #将.deploy目录部署到GitHub
  • hexo help # 查看帮助
  • hexo version #查看Hexo的版本
  • hexo deploy -g #生成加部署
  • hexo server -g #生成加预览
    命令的简写
  • hexo n == hexo new
  • hexo g == hexo generate
  • hexo s == hexo server
  • hexo d == hexo deploy
